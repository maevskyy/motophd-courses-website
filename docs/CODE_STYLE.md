# Code Style — Next.js

Обязательные правила. Нарушение = долг к исправлению, не «стиль на вкус».

## 1. Размер файла

- Компоненты, хуки, логика — **≤ 200 строк** (цель 100–150). 200 — жёсткий максимум.
- SCSS/CSS — **≤ 400 строк**.
- Данные/фикстуры (`lib/content/*`) — дробим по смыслу (курсы, отзывы, FAQ — отдельными файлами), не один свал.
- Упёрся в лимит → **делим**: выносим подкомпоненты, хук (`useX.ts`), утилиты, типы. Не «ужимаем», а разбиваем ответственность.

## 2. Колокация: компонент = папка

Всё про компонент лежит рядом, чтобы папку можно было выдрать и вставить целиком. Не искать «где мой CSS».

```
components/CourseCard/
  CourseCard.tsx          # разметка + минимальная логика
  CourseCard.module.scss  # ЕГО стиль (BEM)
  CourseCard.types.ts     # пропсы/типы, если нетривиальные
  useCourseCard.ts        # хук, если логики много
  index.ts                # ре-экспорт (import из '@/components/CourseCard')
```

- Стиль компонента — **только** в его `*.module.scss`. Общих монстро-стилей не существует.
- Глобально допустимо ровно два файла: `styles/tokens.scss` (CSS-переменные) и `app/globals.scss` (reset/база).

## 3. CSS

- SCSS-модули + BEM (ADR-7).
- Один `*.module.scss` = стиль одного компонента, ≤ 400 строк.
- Файлов уровня `Prototype.module.scss` на 2000 строк — **нет**. Разносим по компонентам.

## 4. Декомпозиция логики

- Один файл — одна ответственность.
- `page.tsx` / `layout.tsx` — **тонкие**: композиция из компонентов, без вёрстки-простыни и без бизнес-логики.
- Бизнес-логика — в `lib/`, не в компонентах.
- Инлайн-стилей и инлайн-обработчиков нет (инвариант).

## 5. Практики Next.js

**Server / Client**
- Server Components по умолчанию; `'use client'` — только на листовых интерактивных компонентах, границу двигаем вниз.
- Данные грузим на сервере (Server Component), не в `useEffect`.

**Структура**
- `page.tsx` / `layout.tsx` тонкие — только композиция.
- Абсолютные импорты через `@/`.
- `loading.tsx` / `error.tsx` / `not-found.tsx` на маршрутах.
- Barrel-`index.ts` умеренно (tree-shaking, циклы импортов).

**Типы**
- Strict TS, ноль `any`; пропсы через `interface Props`.
- Валидация env через zod; `NEXT_PUBLIC_` — только клиентское.

**Производительность**
- `next/image`, `next/font`, `next/link` вместо ручных `<img>`/`<a>`/шрифтов.
- Мемоизация (`memo`/`useMemo`) — только по замеру, не превентивно.
- `<Suspense>`-стриминг где уместно.

**SEO / a11y**
- Metadata API (`generateMetadata`) на маршрутах.
- Семантический HTML, `alt`/labels, управляемый фокус.

**Прочее**
- Тесты рядом с компонентом (`Component.test.tsx`) — когда решим про тесты.
- Именование: PascalCase — компоненты, camelCase — функции/хуки.
- Серверный стейт (данные) ≠ клиентский UI-стейт; глобальный стейт-менеджер пока не нужен.
