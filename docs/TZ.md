## 1. Обзор и бизнес-правила

MotoPhD — образовательная платформа мотокоуча (5000+ офлайн-учеников, YouTube @MotoPhD). Сайт — звено в связке «Я → Мотошкола → YouTube → Курсы». Главная цель — **продавать курсы**. Требование к виду: просто, красиво, продающе; **без** сложного дизайна, длинного скроллинга и пафоса. Референс: cheparacing.com.

**Бизнес-правила (подтверждено брифом):**
- **2 курса на старте**; если пойдёт — до ~10 за год. Контент курса после запуска **фиксированный**.
- Курс — узкая тема, глубоко разобранная. Состав: теория (PDF или PDF+видео) → ключевой поинт → 2–3 видео-упражнения + короткие PDF-инструкции. Итого ~4–5 видео + PDF на курс.
- Тарифы: **«Только курс» ~€29** (точную цену Влад подтвердит) и **«Курс + обратная связь» €129**.
- **Докупка обратной связи** к уже купленному курсу — обязательна (доплата разницы).
- **Обратная связь происходит вне сайта:** ученик и Влад уходят в WhatsApp/Telegram, ученик шлёт видео своей езды, Влад делает разбор + один Zoom 45 минут. Сайт продаёт тариф и выдаёт инструкцию.
- **Промокоды — в MVP** (нужны сразу на старте).
- Доступ к купленному — **навсегда**. Возврата после получения доступа **нет**. Сертификатов **нет**. Бандл «все курсы» — потом.
- Бесплатные **тизеры**: один на главной + по одному на каждый курс (Влад уточнит с маркетологом).
- **Языки EN + RU** (ES позже). На каждый язык — **свои видео и свои PDF**. Выбор языка: авто по браузеру + ручной переключатель.
- Налогообложение — Украина, ФОП → WayForPay. Чек после оплаты уходит покупателю и в налоговую.
- **Срок запуска: строго 01.09.2026.** Ежемесячные расходы держим минимальными.

## 3. Архитектура и стек

Стек и топология — **[ARCHITECTURE.md](./ARCHITECTURE.md)**. Кратко:

- **Next.js (App Router) + TypeScript** — фронт, бэкенд (Route Handlers) и **Payload CMS 3** (админка `/admin`) в одном приложении, один репозиторий.
- **PostgreSQL** self-hosted (контейнер на VPS), `@payloadcms/db-postgres`. Redis не используется.
- **Хостинг:** один VPS (Hetzner) + Docker Compose; Caddy; Cloudflare free (DNS/CDN/DDoS/Access). Один образ → два контейнера: сайт (`/admin*` зарезан) и `admin.motophd.com` за Cloudflare Access.
- **Медиа:** обложки/PDF — Cloudflare R2; видео — Cloudflare Stream (signed URLs).
- **Email:** Resend. **Платежи:** WayForPay + PayPal(?) через абстракцию `PaymentProvider`.
- **CI/CD:** GitHub Actions → образ в GHCR → деплой вручную/кнопкой. **Мониторинг:** Grafana+Prometheus+Loki (тех), GA4+Search Console (бизнес/SEO).
- **Стили:** Sass (SCSS-модули) + BEM.

```
motophd/
├─ app/
│  ├─ (marketing)/[locale]/   # лендинг, страницы курсов, юр (SSG/ISR)
│  ├─ (app)/[locale]/         # кабинет, просмотр курса (SSR, auth)
│  ├─ (payload)/admin/        # админка Payload
│  └─ api/                    # вебхуки платежей, выдача защищённых файлов
├─ payload/collections/       # Users, Courses, Lessons, Media, Purchases, PromoCodes, LegalPages
├─ components/                # дизайн-система (SCSS-модули, BEM)
├─ lib/                       # payments/, access/, i18n/, pdf/, video/
├─ messages/                  # en.json, ru.json (UI-строки)
└─ tests/
```

## 4. Модель контента

**Иерархия плоская: Course → Lessons.** Модулей нет («один курс и есть модуль»).

```
Course (курс = одна боль, узкая тема)
 ├─ цены: standard (~€29), feedback (€129)
 ├─ outcomes[], keyPoint, commonMistakes, whatYouShouldFeel  ← контент-блоки курса
 ├─ тизер-видео (бесплатное превью)
 └─ Lesson[] (порядок свободный, ~4–6 шт.)
     └─ type: video | pdf | text
```

Рекомендуемый шаблон наполнения: урок-теория (PDF [+видео]) → уроки-упражнения (видео + PDF-инструкция). Блоки «Common Mistakes» / «What You Should Feel» / «Outcome» показываются на продающей странице и на странице курса.

## 5. Модель данных (коллекции Payload)

Поля со звёздочкой* — локализуемые (EN/RU-табы в админке). **Важно: локализуются и ассеты** — у видео и PDF своё значение на каждый язык.

| Коллекция | Ключевые поля |
|---|---|
| **Users** | `email`, `password` (Payload auth), `name`, `role` (`student`\|`admin`), `locale`; профиль дозаполняется позже (страна, мотоцикл, опыт — опционально) |
| **Courses** | `slug`, `title`*, `pain`*, `description`*, `cover`*, `priceStandard`, `priceFeedback`, `currency`, `outcomes[]`*, `keyPoint`*, `commonMistakes`*, `whatYouShouldFeel`*, `teaserVideoId`*, `order`, `status` (`draft`\|`published`) |
| **Lessons** | `course` (rel), `order`, `type` (`video`\|`pdf`\|`text`), `title`*, `durationSec`, `streamVideoId`* (Cloudflare Stream, per-locale), `pdf`* (R2-файл, per-locale), `body`*, `isFreePreview` |
| **Media** | загрузки Payload → R2 (обложки, PDF) |
| **Purchases** | `user`, `course`, `tier` (`standard`\|`feedback`\|`feedback_upgrade`), `amount`, `currency`, `provider` (`wayforpay`\|`paypal`\|`mock`\|`manual`), `providerTxnId`, `status` (`pending`\|`paid`\|`failed`\|`refunded`), `promoCode?`, `createdAt`. `manual` — ручная выдача доступа (друг/блогер/бартер) |
| **PromoCodes** (MVP) | `code`, `discountType` (`percent`\|`fixed`), `value`, `maxUses`, `usedCount`, `validFrom`, `validTo`, `active` |
| **LegalPages** | `slug` (`privacy`\|`terms`\|`refund`\|`contact`), `title`*, `body`* |

**Удалено против v1:** `Modules` (структура плоская), `Progress` (прогресс не нужен), `FeedbackRequests` (обратная связь вне сайта).

**Правило доступа:** контент курса доступен при `Purchase{user, course, status:'paid'}`. Feedback-тариф активен при `tier ∈ {feedback, feedback_upgrade}`.

## 7. Функциональные требования (backlog)

### F0 — Инфраструктура и каркас *(Дима, «стартер» — см. `docs/scope-starter.md`)*
VPS + Compose (site, admin, postgres, caddy, мониторинг), CI/CD (образ → GHCR → кнопка деплоя), бэкап-крон, Cloudflare (DNS/CDN/Access), R2 + Stream подключены, скелет Next+Payload, тест-раннеры, reference feature, README для Ани.
**Приёмка:** прод-смоук пройден; Аня может поднять проект локально по README и задеплоить кнопкой.

### F1 — Дизайн-система
Токены из прототипа (тёмная тема, красный `#E31E24`), базовые компоненты (Button, Card, Section, Nav, Footer, FormField, Accordion, Toast). SCSS-модули + BEM.
**Приёмка:** компоненты соответствуют прототипу; состояния hover/active/disabled.
**Тесты:** component-рендер.

### F2 — i18n + Лендинг
Локали EN/RU, словари, авто-детект + переключатель. Секции: hero, авторитет-бар, 2 курса, метод, how-it-works, о тренере, финальный CTA, футер. **Компактно — Влад против длинного скролла** (секции отзывов из прототипа НЕ переносим; FAQ — под вопросом, §16). Тизер-видео на главной. SEO-метаданные, hreflang, sitemap.
**Приёмка:** обе локали работают; Lighthouse SEO ≥ 95; контент из Payload.
**Тесты:** e2e переключения языка; unit hreflang/sitemap.

### F3 — Страницы курсов ×2
Роут `/[locale]/courses/[slug]`: hero + боль, outcomes, key point, состав курса (уроки), Common Mistakes / What You Should Feel, тизер, sticky-блок с тарифами (€29 / €129) + чекбоксы (§10) + промокод, CTA. Каталог: при 2 курсах — секция главной, отдельная страница появится при росте (§16).
**Приёмка:** данные из Payload; «Купить» неактивна без чекбоксов; SSG/ISR.
**Тесты:** component (тарифы, чекбоксы, промокод-поле); e2e переход к оплате.

### F4 — Авторизация
Email + пароль (Payload auth). **Покупка гостем без предварительной регистрации** — аккаунт создаётся автоматически после оплаты, креды приходят на почту; возможность смены пароля. Вход/выход, восстановление пароля. **Без подтверждения email.** Данные при регистрации — минимум (email); профиль дозаполняется в кабинете.
**Приёмка:** приватные роуты закрыты; письмо с доступами приходит; восстановление работает.
**Тесты:** unit валидации; e2e логин/логаут/восстановление.

### F5 — Контент-модель + Админка
Коллекции §5, локализация полей и ассетов, загрузка PDF → R2, привязка Stream-видео, статусы draft/published, русские подписи полей. Админ-функции: CRUD курсов/уроков, цены, промокоды, покупки, **ручная выдача доступа** (`provider:'manual'`), статистика продаж (список/сумма Purchases), тексты LegalPages.
**Приёмка:** Аня создаёт курс с уроками на двух языках и публикует без программиста; студент в `/admin` не попадает.
**Тесты:** integration (Local API CRUD); access-control.

### F6 — Личный кабинет
Мои курсы (купленные + не купленные с CTA), профиль (имя, email + дозаполнение), смена пароля, история покупок/чеки, **удаление аккаунта**, докупка feedback-тарифа. **Без прогресса.**
**Приёмка:** видно только своё; удаление аккаунта работает (с подтверждением); докупка ведёт на оплату разницы.
**Тесты:** e2e кабинета; access (чужие покупки недоступны).

### F7 — Просмотр курса + защита контента
Страница курса для купившего: список уроков (свободный порядок, без отметок), видео-плеер (Stream, signed playback-токен с сервера), PDF (через приложение, с вотермарком email; режим просмотр/скачивание — §16). Серверная проверка покупки на каждый ресурс.
**Приёмка:** без покупки видео/PDF недоступны (403), прямых ссылок нет; тизеры доступны всем.
**Тесты:** e2e доступа (купил/не купил); unit (signed URL, вотермарк).

### F8 — Checkout + платежи + промокоды
Абстракция `PaymentProvider` (wayforpay / paypal / mock). WayForPay: карта + Apple Pay/Google Pay; PayPal — под вопросом (§16). Флоу: выбор тарифа → чекбоксы → [промокод] → оплата → server-to-server колбэк с проверкой подписи → `Purchase:paid` → аккаунт+письмо. **Промокоды в MVP** (валидация и расчёт на сервере). **Докупка feedback = оплата разницы** (`feedback_upgrade`, €100 при 129/29). Идемпотентность колбэков. Цены в EUR (автовалюта — post-MVP). Fallback-блок «не получается оплатить — свяжитесь с нами».
**Приёмка:** успешная оплата открывает доступ; повторный колбэк не дублирует; промокод меняет цену корректно; апгрейд работает.
**Тесты:** unit (цены, промокоды, апгрейд, подпись, идемпотентность); e2e на mock/sandbox.
**Внешний блокер:** ФОП + мерчант WayForPay ещё не оформлены — задача Влада №1.

### F9 — Feedback-тариф (пост-покупка) — *сильно упрощено против v1*
Никаких форм и переписки на сайте. После покупки тарифа `feedback`/`feedback_upgrade`: страница + письмо с инструкцией (контакт WhatsApp/Telegram, что прислать, условия: 1 видео-разбор + 1 Zoom 45 мин, срок ответа — зафиксировать в Terms, §16). В админке — фильтр покупателей feedback-тарифа.
**Приёмка:** инструкцию видит только купивший тариф; письмо приходит.
**Тесты:** access + e2e письма.

### F10 — Юр, consent, аналитика
LegalPages (Privacy/Terms/Refund/Contact — шаблон, Влад дорабатывает + юрист). Два обязательных чекбокса на покупке (§10). Cookie-баннер (GDPR), GA4 + Meta Pixel через Consent Mode (после согласия), Search Console, copyright.
**Приёмка:** трекинг не грузится до согласия; чекбоксы обязательны; тексты локализованы.
**Тесты:** e2e consent-флоу.

### F11 — Финальный адаптив + QA
Mobile-first ревизия всех экранов (прототип имеет один брейкпоинт — расширить), перформанс, a11y, кросс-браузерность.
**Приёмка:** нет горизонтального скролла; тач-цели ≥44px; Lighthouse mobile Perf ≥ 85, A11y ≥ 95.

### Post-MVP
Бандл «все курсы», маркетинговые рассылки (согласие собираем с MVP), отзывы (Влад: «потом сами»), ES-локаль, автовалюта по стране, ТГ-комьюнити для RU-покупателей, FAQ-страница.

## 8. Платежи (детали)

- Интерфейс `PaymentProvider`: `createCheckout(order)` / `verifyCallback(payload)`; реализации wayforpay, paypal, mock.
- WayForPay: HMAC-подпись колбэка (`merchantSignature`), `orderReference`, статусы. Apple/Google Pay — через WayForPay.
- Расчёт цены **только на сервере**: тариф → промокод → итог. Апгрейд: `priceFeedback − priceStandard`.
- Идемпотентность: повторный колбэк по тому же `orderReference` не создаёт вторую покупку.
- Валюта EUR. Автоопределение по стране — post-MVP (§16).
- **Блокер:** ФОП и боевой мерчант — на Владе; разработка идёт на mock + sandbox.

## 9. Защита контента

- **9.1 Видео — Cloudflare Stream:** подписанные playback-URL (`requireSignedURLs`) с коротким TTL, HLS, скачивание выключено, allowed origins. Токен генерируется на сервере после проверки покупки. Стоимость ~$5/1000 мин хранения + $1/1000 мин доставки. Фолбэк — Bunny Stream.
- **9.2 PDF:** приватный бакет R2, выдача только через приложение: проверка покупки → вотермарк (email + номер заказа, `pdf-lib`) → короткоживущая ссылка. Режим (просмотр/скачивание) — §16.
- **9.3:** любой защищённый ресурс = серверная проверка `Purchase:paid`, иначе 403. Уровень защиты — базовый (по брифу), без DRM.

## 10. Юридическое

CI на каждом PR; **фича не мержится без тестов на её приёмку.**

## 14. DevOps

- **Окружения:** local (docker compose) → production (VPS). Постоянного dev-стенда нет; при необходимости — staging-контейнер на том же VPS.
- **Хостинг:** один VPS (Hetzner, ~8 ГБ) + Docker Compose: app-site, app-admin (тот же образ), postgres, caddy, grafana, prometheus, loki (+exporters).
- **CI/CD:** GitHub Actions — lint/тесты на PR; сборка образа → GHCR; деплой вручную (Run workflow). Rollback = деплой предыдущего тега.
- **БД:** Postgres в контейнере; ночной `pg_dump` → offsite (Hetzner Storage Box / B2) по правилу 3-2-1; восстановление периодически тестируется.
- **Медиа:** R2 (обложки/PDF) и Cloudflare Stream (видео) — managed-durability, вне бэкап-скоупа.
- **Секреты:** env-переменные на VPS и в CI. **Миграции:** Payload/Drizzle, версионируются.
- **Мониторинг:** Grafana/Prometheus/Loki + аптайм-чек; Sentry опционально.
