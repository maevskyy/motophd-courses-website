**Goal:** Всё содержимое `motophd-demo.html` (1319 строк: 6 экранов + стили + переводы) живёт в Next-приложении как нормальные компоненты. **Пока без изменений** — перенос как есть, упрощения по ТЗ v2 будут отдельными задачами.

## Scope (черновик — уточним на груминге)

- Страницы: главная, каталог, страница курса, логин, кабинет, просмотр курса — в route-группы по ARCHITECTURE.md.
- CSS из `<style>` → SCSS-модули по компонентам, классы → BEM. Токены (`--red` и пр.) → общие переменные.
- Словарь `T{}` + `HTML_OVERRIDES` из скрипта → `messages/en.json` / `ru.json`; переключатель языка → роутинг локалей (next-intl), а не JS-подмена текста.
- Интерактив (FAQ-аккордеон, табы, curriculum, мобильное меню) → React-состояние. Никакого `onclick=""` в разметке.
- Данные (курсы, уроки, отзывы) — пока **хардкод** в константах, НЕ Payload (подключение данных — отдельная задача).
- Разбивка на компоненты дизайн-системы: Button, Card, Section, Nav, Footer, Accordion, FormField, Toast.

## Acceptance criteria (черновик)

- Все 6 экранов пиксельно соответствуют прототипу в обеих локалях; `motophd-demo.html` можно открыть рядом и не найти отличий на десктопе.
- Ни одного инлайн-стиля и инлайн-обработчика; каждый экран собран из компонентов.
- Существующий брейкпоинт 900px воспроизведён (полный адаптив — позже, F11).
- Lint/тесты зелёные; смоук-e2e: каждая страница рендерится, переключение локали работает.

## Out of scope

- Упрощения ТЗ v2 (удаление прогресса/модулей/отзывов, 2 курса, €129, промокод-поле, второй чекбокс) — отдельные задачи после переноса.
- Подключение к Payload, реальные auth/оплата/видео.
