# Решения (ADR)

Формат: контекст → решение → следствия. Пересматривать решение = новая запись, старую не стирать.

## ADR-1 — Next.js + Payload в одном приложении (не два)
Payload 3 живёт внутри Next: Local API без HTTP, общая auth, одна кодовая база — критично для Ани.
Изоляция админки — не сплитом, а **вторым контейнером из того же образа**: публичный контейнер режет `/admin*` на прокси, `admin.motophd.com` — за Cloudflare Access.
*Следствие:* монорепы и REST-прослойки нет; изменения схемы сразу видны сайту.

## ADR-2 — Один VPS вместо Vercel/Neon/R2-serverless
Причины: страх variable-биллинга (Vercel), предсказуемая цена, единая точка дебага, бэкендерский опыт Димы. Worst-case — даунтайм, а не счёт на $20k.
*Следствие:* сами отвечаем за бэкапы (3-2-1, offsite) и обновления; Cloudflare-прокси обязателен (DDoS). Обоснование и источники — в истории проекта (12-factor, 37signals cloud exit).

## ADR-3 — Cloudflare-консолидация: R2 + Stream (не Bunny)
Cloudflare неизбежен (CDN/DNS/DDoS/Access) ⇒ R2 и Stream не добавляют вендора.
Видео с VPS не раздаём никогда (egress, транскодинг, защита). R2 — durable, медиа вне бэкап-скоупа.
*Следствие:* «одна консоль — четыре роли»; fallback при проблемах Stream — Bunny.

## ADR-4 — Без Redis
Сессии — JWT в httpOnly-куке (stateless), очередей в MVP нет, кэш — Cloudflare.
*Следствие:* добавляем только при появлении фоновых задач (рассылки — post-MVP).

## ADR-5 — CI/CD без деплой-панелей
GitHub Actions: PR-чеки → образ в GHCR → деплой кнопкой `Run workflow`; rollback = предыдущий тег. Kamal/Coolify отвергнуты — лишний слой, Дима деплоит сам / Аня кнопкой.
*Следствие:* образ собирается в CI, не на VPS (память); healthcheck после выкатки обязателен.

## ADR-6 — Два дома для текстов (это не дублирование)
UI-строки (кнопки, формы, ошибки) — `messages/*.json`, живут с кодом, меняются деплоем.
Контент (курсы, маркетинговые и юр-тексты) — Payload, per-locale, меняется без деплоя, **включая файлы** (у EN и RU свои видео/PDF).
Лакмус: «нужен ли деплой, чтобы это изменить?» Excel-таблиц переводов нет.

## ADR-7 — Sass (SCSS-модули) + BEM
Решение Димы, не пересматривается. SCSS встроен в Next, модули изолируют классы. Tailwind и UI-киты не подключаем.

## ADR-8 — Метрики не смешиваются
Grafana/Prometheus/Loki — только техздоровье. GA4 (+Pixel по решению маркетолога) — бизнес. Search Console — SEO. Продажи — админка Payload. Бизнес-метрики в Grafana не тащим.
